# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Degree Map — a **client-only** degree planner for UBC students. Search the
real 7,269-course catalog, place courses on a year × term board, get live
prerequisite validation and degree progress. No backend: static data + pure
TypeScript engine + React UI. Plans persist to localStorage and share via
URL fragment. See PLAN.md for the full design rationale (this repo is a v2
rewrite that deliberately retired an Express/Redis/BullMQ backend).

## Commands

```bash
npm run dev       # Vite dev server at :5173
npm run data      # regenerate public/data/ from data/source/ (tsx)
npm run test      # vitest run (engine + parser unit tests)
npx vitest run src/engine/validate.test.ts   # single file
npm run build     # data + tsc -b + vite build → dist/
npm run preview   # serve dist/ at :4173
npm run e2e       # browser smoke test (needs preview running; playwright-core)
npm run lint      # typecheck only
```

## Architecture — layering is the contract

1. **`src/engine/`** — THE CORE. Pure functions over plain data, zero I/O,
   zero React, fully unit-tested. `types.ts` (domain types), `prereq.ts`
   (rule-tree evaluation with per-node met/unmet annotation), `validate.ts`
   (whole-plan validation + per-term eligibility), `progress.ts`
   (requirement matching + degree progress). **Never import React, fetch,
   or the store here.** New domain logic goes here, with tests.
2. **`src/ai/`** — the AI advisor layer: pure logic + fetch-only providers,
   no React, no store imports. `context.ts` (grounding: serializes plan +
   engine results + ranked eligible-candidate pool into a small-model-sized
   prompt), `parse.ts` (lenient reply parser — prose + fenced-JSON
   recommendations, never throws), `providers/` (mock heuristics by default;
   OpenAI-compatible + Anthropic BYOK behind `provider.ts`). The advisor
   conversation lives on `Plan.advisor` (duplicated/exported with the plan,
   **stripped from share URLs**); provider settings live in a separate
   localStorage key (`degree-map-ai`, `src/state/aiSettings.ts`) so keys
   never leak through export/share.
3. **`src/catalog/loader.ts`** — data access: loads `public/data/index.json`
   upfront, lazy-loads `public/data/dept/XXXX.json` chunks, module-level
   in-flight dedup. Search ranking also lives here.
4. **`src/state/store.ts`** — Zustand store. Persisted slice = `{plans,
   activePlanId}` (version 3; `migratePersisted` backfills older shapes).
   Session slice = catalog cache + UI state. Share links = plan minus
   `advisor`, lz-string-compressed into `#plan=`.
5. **`src/ui/`** — components stay thin: render state, dispatch actions,
   display engine results. Validation runs synchronously in `App.tsx` via
   `useMemo` on every plan/catalog change.
6. **`scripts/build-data.ts`** — build-time pipeline: `data/source/*.json`
   (scraped catalog + programs, checked in) → `public/data/` (gitignored,
   regenerated by `npm run data`, which `npm run build` runs automatically).
   The prereq-prose parser lives in `scripts/lib/parsePrereq.ts` and only
   runs at build time.

## Domain semantics (encoded in engine tests — don't change casually)

- Statuses: `planned | in_progress | completed | failed`. Everything except
  `failed` counts as "taken" for prereqs/progress; planning assumes success.
- Prereqs must be satisfied by **strictly earlier** terms; coreqs by the
  same term or earlier. Completed/failed entries are history and not
  re-validated.
- Prose-only prerequisites ("third-year standing…") are **never guessed**:
  the pipeline keeps them as `prereqText`, the engine reports
  `prereq_unknown` as a warning, the UI shows the prose verbatim.
- `unlocks` = reverse edges of course leaves in prereq trees (min_credits
  pools excluded), precomputed by the pipeline.
- Term credit-load warnings: 18 (winter terms), 9 (summer).
- Duplicate course = error, unless the earlier attempt is `failed` (retake).

## Gotchas

- `public/data/` is generated — don't edit it; edit `data/source/` or the
  pipeline and re-run `npm run data`.
- Program requirement specs: only `cs-major` has real requirements in
  `data/source/programs.json`; the pipeline drops requirement-less programs
  and folds faculty-level requirements into each program's list.
- The search index (`CourseLite`) is a tuple array to stay small — use the
  `lite*` accessors from `src/engine/types.ts`, don't index positions
  directly.
- Status/eligibility colors are never the only channel — every status has a
  distinct glyph (`src/ui/bits.tsx`) and a title/label. Keep it that way.
- `npm run e2e` finds a browser via `$CHROME_PATH`, then the ms-playwright
  cache, then the Edge channel; it clears the app's localStorage key first.
- Windows dev machine: Node was installed via `winget --scope user`; if a
  fresh shell can't find `node`, prepend the User PATH
  (`[Environment]::GetEnvironmentVariable('Path','User')`).
